## 2.1 Docker — M — Sonnet 5

- Backend `Dockerfile`: multi-stage (builder + slim runtime), uvicorn, non-root user. Note: WeasyPrint needs Pango system libs in the runtime image.
- Frontend build stage (static output; served by Static Web Apps in prod, nginx or vite preview in compose).
- `docker-compose.yml` for local dev (backend + frontend; DB stays Azure SQL dev).

**Acceptance:** `docker compose up` runs the stack locally against dev Azure resources.

## 2.2 Bicep per-client stack — L — Fable 5 design → Opus 4.8 implement

`infra/bicep/main.bicep` + one params file per client. Provisions, in one deployment:

- Resource group (tagged `client=<name>` for cost tracking)
- Azure SQL (S0), private-endpoint-ready
- Storage account (blob soft-delete + versioning on, container `pliegos-raw`)
- AI Search (Basic)
- Document Intelligence
- Azure OpenAI — **Sweden Central or Spain Central** (EU data residency is the sales pitch); deployments: chat + embeddings, names parameterized
- Key Vault (RBAC mode; only JWT-SECRET + SQL-CONNECTION as secrets after task 1.4)
- Application Insights + Log Analytics
- Container Apps env + app (backend), system-assigned Managed Identity
- Static Web App (frontend)
- RBAC role assignments for the Managed Identity (Search Index Data Contributor, Storage Blob Data Contributor, Cognitive Services OpenAI User, etc.)

**Acceptance:** `az deployment sub create` on a clean subscription → working environment with zero manual portal steps.

## 2.3 CI/CD — M — Sonnet 5

GitHub Actions:
- PR: ruff + pytest (backend), typecheck + build + vitest (frontend).
- Deploy: manual `workflow_dispatch` with environment input (client name) → build image → push to registry → update Container App; deploy frontend to SWA. Alembic upgrade step (after 2.4).

**Acceptance:** green PR pipeline; one-click deploy to a named client env.

## 2.4 ♻ Alembic — M — Opus 4.8

- Baseline autogenerated from current models; stamp existing dev DB.
- Runs in the deploy workflow before app rollout.

**Acceptance:** `alembic upgrade head` idempotent on fresh and existing DBs.

## 2.5 Admin CLI — S — Haiku 4.5

`python -m app.cli create-org-admin --email ... --name ...` against any env (uses env's DB). Replaces the dev-only `/register` for provisioning the first client user. Generates a temp password or invite token (ties into 3.3 later).

## 2.6 Cost guardrails — S — Sonnet 5

Budget + alert (email) per client subscription/RG in Bicep; consistent resource tags. **Acceptance:** threshold alert fires on test budget.
